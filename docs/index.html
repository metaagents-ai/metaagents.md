MetaAgents

An open standard for composing agents from reusable skills and MCPs — with dependency resolution built in.

Read the Spec GitHub

Why MetaAgents?

Existing standards define agents and skills in isolation. MetaAgents adds the missing piece: how they compose together.

  • AGENTS.md tells an agent what to do
  • Agent Skills packages a single capability
  • MetaAgents declares how agents, skills, and MCPs depend on each other

Core Concepts

Agent

An execution template that combines instructions with skills and MCPs.

AGENTS.md

Skill

A reusable capability package — methodology, knowledge, or workflow.

SKILL.md

MCP

A Model Context Protocol server configuration. Public servers are listed in the official MCP registry.

<name>.json

How It Works

Add a dependencies field to your frontmatter. That's it.

---
name: langsensei/code-reviewer
description: Reviews PRs with security and style checks.
version: 1.0.0
dependencies:
  skills:
    - langsensei/security-audit
    - style-guide
  mcps:
    - io.github.modelcontextprotocol/filesystem
    - filesystem
---

## Instructions

Review the PR for security vulnerabilities...

MCP names follow the validation rules of the official MCP registry — mixed case, underscores, and longer identifiers (up to 200 chars) are all permitted. Both io.github.modelcontextprotocol/filesystem (registry-canonical reverse-DNS) and filesystem (unscoped local) are valid references.

Key Features

  • Backward compatible — existing AGENTS.md and SKILL.md files work without changes
  • Scoped names — scope/name format prevents conflicts across sources, and matches the official MCP registry convention
  • Dependency graph — topological resolution with cycle detection
  • Prerequisites — skills declare setup steps the LLM verifies after install
  • Format-only — no opinions on package managers, runtimes, or execution

Get Started

MetaAgents is just YAML frontmatter in your existing markdown files. No new tools required.

# 1. Add dependencies to your AGENTS.md
---
name: my-agent
description: Does useful things.
dependencies:
  skills:
    - langsensei/some-skill
  mcps:
    - some-mcp
---

# 2. Your skill declares its own deps
---
name: langsensei/some-skill
description: A reusable capability.
dependencies:
  mcps:
    - another-mcp
prereqs: "Run 'pip install x' first. Read references/setup.md for details."
---

Ecosystem

MetaAgents is designed to layer on top of existing community standards rather than replace them:

  • agents.md — the underlying instruction format for agents
  • Agent Skills — the underlying skill packaging format
  • Model Context Protocol — the underlying tool-server protocol for MCPs
  • MCP Registry — the official directory of public MCP servers; MetaAgents reuses its scope/name identifier convention

Tools that don't understand MetaAgents simply ignore the extra frontmatter fields. Tools that do can resolve dependency graphs, install transitively, and verify prerequisites.
